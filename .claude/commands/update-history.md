# 개발 히스토리 업데이트

이번 대화에서 수행한 개발 작업을 날짜별로 요약하여 히스토리 파일에 기록합니다.

## 입력
- $ARGUMENTS: (선택) 추가 메모 또는 특이사항. 비어있으면 자동으로 현재 대화 내용 기반 요약.

## 수행 작업

### 1. 변경사항 수집
다음 소스를 확인하여 이번 세션에서 변경된 내용을 파악한다:

- `git diff --stat` — 변경/추가/삭제된 파일 목록
- `git diff --name-status` — 파일별 상태 (A/M/D)
- `git log --oneline -10` — 최근 커밋 메시지
- 현재 대화에서 수행한 작업 내역

### 2. 히스토리 파일 업데이트
`fnco_influencer_seeding-main/tasks/history.md` 파일에 아래 형식으로 **맨 위에** 추가한다 (최신이 위):

```markdown
## YYYY-MM-DD (요일)

### 작업 요약
- 한 줄 요약 (예: "AI-PLAN 워크플로우 Phase 3 연동 완료")

### 변경 내역
| 구분 | 파일/모듈 | 내용 |
|------|-----------|------|
| NEW | `경로/파일명` | 신규 생성 설명 |
| MOD | `경로/파일명` | 수정 내용 설명 |
| FIX | `경로/파일명` | 버그 수정 설명 |
| DEL | `경로/파일명` | 삭제 사유 (해당 시) |

### 주요 결정사항
- 기술적 결정이나 설계 선택 기록 (있을 때만)

### 참고
- 관련 이슈, 미완료 작업, 다음 세션 TODO (있을 때만)

---
```

### 3. 파일이 없으면 생성
`history.md`가 없으면 아래 헤더로 새로 생성:

```markdown
# FNCO Influencer Seeding — 개발 히스토리

> 날짜별 개발 작업 기록. 최신 항목이 맨 위.

---
```

## 분류 기준
- **NEW**: 새 파일/컴포넌트/API 생성
- **MOD**: 기존 파일 수정 (기능 추가, 로직 변경)
- **FIX**: 버그 수정
- **DEL**: 파일 삭제 (사용자 승인 받은 경우만)
- **CFG**: 설정/환경변수/의존성 변경

## 규칙
- 최신 날짜가 파일 맨 위에 오도록 역순 정렬
- 변경 내역은 구체적으로 (파일 경로 + 무엇을 했는지)
- 사소한 변경(포맷팅, 주석)은 생략
- 기존 히스토리 항목을 수정/삭제하지 않는다
- git commit 전에도 실행 가능 (unstaged 변경 포함)
